# Target Technology Stack

This record owns Musubi's selected technologies, the few non-default restrictions needed to use them together, and the conditions that would justify reconsidering a selection. It does not own product behavior, architecture, migration steps, or acceptance procedures.

## Selected technologies

| VueUse only for a named concrete need                                                                                            | A proven composable may avoid maintaining generic browser or reactivity behavior, while unused abstractions do not enter the project.                                                                                      | Add an individual VueUse capability only when its concrete caller and benefit are identified; remove the dependency if no selected use remains. |
| pnpm                                                                                                                             | It is Musubi's selected package manager and works with the selected Vite+ workflow.                                                                                                                                        | A required environment cannot support it or Yunfei explicitly changes the package-manager choice.                                               |

Nuxt lifecycle commands still run through Nuxt package scripts invoked by `vp run`.                                                                             | Its Nuxt-safe use becomes unmaintainable or it no longer supplies a required part of the selected toolchain.                                    |
| Upstream Vite inside Nuxt, never a Vite+ replacement or alias                                                                    | Nuxt and its modules must use the upstream Vite selected by Nuxt; Vite+ remains the surrounding toolchain rather than replacing Nuxt's builder.                                                                            | Nuxt officially supports and tests the replacement, or the replacement becomes upstream Vite itself.                                            |
| All Vite Task and `vp run` result caching disabled                                                                               | Repeated iteration must execute against the current content, environment, and code rather than restore a prior result. Set root `run.cache: false`, set `cache: false` on every declared task, and never enable `--cache`. | Yunfei explicitly reopens the rule after a measured need and accepts a reviewed cache-correctness model.                                        |
| Official `@notionhq/client`                                                                                                      | It is the maintained client for the selected Notion APIs; architecture owns how Musubi uses those APIs.                                                                                                                    | The official client cannot support a required current Notion capability or creates a demonstrated build-time blocker.                           |
| Unified with `remark-parse`, `remark-gfm`, and syntax-only `remark-mdx`                                                          | These parsers cover the selected Markdown syntax and expose an inspectable tree without selecting an executable MDX runtime.                                                                                               | The accepted Notion Markdown cannot be represented safely and maintainably, or a smaller maintained parser proves complete coverage.            |
| `subset-font` and `fonteditor-core`                                                                                              | They provide the selected Node and WebAssembly font subsetting and WOFF2 identity-editing capabilities without a separately installed native toolchain.                                                                    | They cannot preserve the required font data, licensing metadata, deterministic output, or browser-valid WOFF2 result.                           |
